# /gh:debt - 技術的負債管理コマンド

> **核心**: スコープ外で却下/スキップした指摘を追跡し、忘れ去られることを防ぐ

## Triggers

- 定期的な負債確認（月次推奨）
- 新機能開発前の負債チェック
- /gh:review 完了後の累計確認
- 負債件数が閾値（20件）を超過した時

## Usage

```bash
# 負債一覧表示
/gh:debt

# 重要度でフィルタ
/gh:debt --severity critical,high

# 特定ファイルの負債
/gh:debt --file src/auth.py

# Issue化（選択式）
/gh:debt --create-issues

# 古い負債をクリーンアップ（対応済み確認）
/gh:debt --cleanup
```

## Behavioral Flow

### Phase 1: Load Technical Debt

```yaml
1. claudedocs/technical_debt.md 読み込み:
   - 却下された指摘
   - スキップされた指摘
   - 追加日時、元PR番号

2. 統計計算:
   - 総件数
   - 重要度別内訳
   - 最古の未対応日
   - ファイル別集計
```

### Phase 3: Actions (Interactive)

```yaml
5. アクション選択:

   AskUserQuestion:
     "📋 次のアクションを選択:
     
      1. Issue化する（推奨: Critical/High を Issue に変換）
      2. 対応済み確認（ファイルを検査して解決済みを削除）
      3. 一括却下（本当に不要な項目を削除）
      4. 終了"

6. Issue化フロー（選択1）:

   for debt in selected_debts:
     gh issue create \
       --title "Tech debt: [debt.summary]" \
       --body "$(cat <<EOF
   ## 元の指摘
   
   - **ファイル**: ${debt.file}:${debt.line}
   - **重要度**: ${debt.severity}
   - **元PR**: #${debt.pr_number}
   - **却下理由**: ${debt.reject_reason}
   
   ## 指摘内容
   
   ${debt.body}
   
   ## 対応方針
   
   TBD
   
   ---
   Generated by /gh:debt
   EOF
   )"

   # 作成後、technical_debt.md を更新（Issue番号を追記）

7. クリーンアップフロー（選択2）:

   for debt in debts:
     # ファイルを読み込み、指摘箇所を確認
     current_code = read_file(debt.file, debt.line)
     
     # 修正済みか判定
     if is_fixed(debt, current_code):
       mark_as_resolved(debt)
       print(f"✅ 解決済み: {debt.file}:{debt.line}")
     else:
       print(f"⏳ 未解決: {debt.file}:{debt.line}")

8. 完了サマリー:

   ┌──────────────────────────────────────────────────┐
   │ ✅ 技術的負債管理完了                           │
   │                                                  │
   │ Issue化: 3件 → #45, #46, #47                    │
   │ 解決済み削除: 5件                               │
   │ 残り: 15件                                       │
   └──────────────────────────────────────────────────┘
```

## Options

```bash
--severity <levels>   # critical,high,medium,low でフィルタ
--file <path>         # 特定ファイルの負債のみ
--since <date>        # 指定日以降の負債のみ
--create-issues       # 選択した負債をIssue化
--cleanup             # 対応済み確認モード
--export              # CSV/JSON でエクスポート
```

## Integration Points

```yaml
/gh:review:
  - Phase 3.5 で自動追記
  - 却下/スキップ時に記録

/gh:start:
  - 作業開始時に負債件数表示
  - 関連ファイルの負債を警告

/gh:issue close:
  - Issue完了時に関連負債を確認
```

## Thresholds

```yaml
⚠️ 警告レベル:
  - 20件超過: 黄色警告
  - 30件超過: 赤色警告、Issue化を強く推奨
  - 50件超過: 🚨 作業ブロック検討

🔴 Critical 負債:
  - 30日以上未対応: 警告
  - 60日以上未対応: 強制Issue化を提案
```

## 実行指示

**あなたは今、`/gh:debt` コマンドを実行しています。**

### 必須実行フロー

```yaml
1. claudedocs/technical_debt.md を読み込む
   - ファイルが存在しない場合は空の状態として扱う

2. 統計とサマリーを表示

3. AskUserQuestion でアクション選択

4. 選択に応じて処理実行:
   - Issue化: gh issue create
   - クリーンアップ: ファイル検査 → 解決済み削除
   - 一括却下: 確認後削除

5. technical_debt.md を更新

6. 完了サマリー表示
```
