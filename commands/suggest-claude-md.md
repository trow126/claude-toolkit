# /suggest-claude-md - CLAUDE.md改善提案生成

## Purpose
現在の会話を分析し、作業パターン抽出と知識ベース更新のためのCLAUDE.md改善提案を生成します。

## When to Use
- 重要な会話セッションの終了時
- 新しい技術的発見や失敗パターンを記録したい時
- CLAUDE.mdの改善機会を定期的に確認したい時
- Hook実行を待たずに即座に分析したい時

## Analysis Focus

### 1. 作業パターン抽出
以下の会話内容から繰り返しパターンと改善機会を識別してください：

- **繰り返し発生している操作や質問**
  - 同じタイプの質問が複数回出現していないか
  - 毎回説明が必要な概念やプロセスはないか

- **非効率な作業フロー**
  - 回り道をした作業手順
  - より効率的にできた可能性のあるアプローチ

- **自動化可能なタスクパターン**
  - スクリプト化できる手作業
  - スラッシュコマンド化できる定型作業

- **頻出する技術的課題**
  - 繰り返し発生するエラーパターン
  - 特定の技術領域での理解不足

### 2. 知識ベース更新
以下の観点からCLAUDE.mdに追加すべき内容を抽出してください：

- **プロジェクト固有の学習内容**
  - このプロジェクト特有の制約や要件
  - 使用しているフレームワーク/ライブラリの特殊な使い方

- **新しく発見されたベストプラクティス**
  - 今回の会話で効果的だったアプローチ
  - 推奨される実装パターン

- **技術的な意思決定と根拠**
  - 重要な技術選択の理由
  - トレードオフの考察

- **回避すべき失敗パターン**
  - 今回発生したエラーとその原因
  - やってはいけないアンチパターン

## Output Format

以下の形式で改善提案を生成してください：

```markdown
# CLAUDE.md改善提案

## 📊 会話サマリー
- 主要トピック: [会話の主題]
- 完了タスク: [実施した作業]
- 発生した課題: [遭遇した問題]

## 🔄 抽出された作業パターン

### 繰り返しパターン
[何度も発生した操作や質問]

### 改善機会
[より効率化できる可能性のある作業]

### 自動化候補
[スクリプトやコマンド化できるタスク]

## 📚 知識ベース更新候補

### [該当するファイル名: 例えばRULES.md、MODE_*.md、新規ファイル]

**追加すべきセクション:**
```markdown
[CLAUDE.mdファイルに追加する実際のマークダウン内容]
```

**追加理由:**
[なぜこの内容を追加すべきか]

**影響範囲:**
[この追加により改善される作業領域]

## 🎯 優先度と実装順序

### 🔴 高優先度（即座に対応すべき項目）
1. [項目] - [理由]

### 🟡 中優先度（改善推奨項目）
1. [項目] - [理由]

### 🟢 低優先度（参考情報）
1. [項目] - [理由]

## 💡 追加の提案
[その他の改善アイデアや考慮事項]
```

## Instructions

1. **会話全体を分析**: このセッションの全メッセージを振り返ってください
2. **パターンを識別**: 繰り返しや非効率性を見つけてください
3. **具体的な提案**: 抽象的な提案ではなく、実際にCLAUDE.mdに追加できるマークダウン形式で提案してください
4. **優先順位付け**: 影響度と緊急度に基づいて優先順位をつけてください
5. **実装可能性**: 提案は実際に実装可能で、明確な改善効果があるものに絞ってください

## Important Notes

- **具体性重視**: 「認証を改善すべき」ではなく「RULES.mdに認証実装チェックリストを追加」のように具体的に
- **マークダウン形式**: 提案内容は実際にファイルにコピペできる形式で記載
- **影響分析**: 各提案がどのような作業を改善するか明確に説明
- **優先度の根拠**: なぜその優先度なのか理由を示す

それでは、この会話セッションを分析してCLAUDE.md改善提案を生成してください。
